Options

Toolname accepts paramters in two formats: through the command line or through a file config.txt (a sample file is included in the source from GitHub) located in the same directory as the executable. Parameters supplied through the command line take precedence over parameters from config.txt. Commands are always specified as a key-value-pair. Some commands are mandatory, while other have default values.

Parameters in config.txt are specified like this:

% Any line starting with '%' is a comment
% The following lines will assign 'value1' and 'value2' to the parameters 'key1' and 'key2'
key1=value1
key2=value2
A sample config.txt is supplied with the project.

The command line accepts the same parameters as config.txt. To achieve the same result as above, the following is used:

$ toolname -key1 value1 -key2 value2

Protocol format

The protocol file consists of three parts: a declaration of available states, an initial configuration and a set of rules, in that order. Any line starting with a '%' or any empty line is ignored.

States

The state declaration begins with a line with the text "States:". Each following line declares the name of a state, until the starting line of the initial configuration. A valid state name is a string consisting of (A-Z, a-z, 0-9, _).
